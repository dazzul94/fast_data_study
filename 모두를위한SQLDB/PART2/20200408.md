### Data Analytics Value Chain(데이터 분석 가치 사슬)
- 데이터 기반 의사결정을 위한 가치 사슬
- 데이터 분석으로 Insight 도출과 실행으로 가치를 만드는 것이 핵심임

### 데이터 전처리 정의(Data preparation, Data pre-processing)
- 원시 데이터(Raw Data)를 데이터 분석이 가능한 형태로 만드는 과정이다.
- 그래서, 데이터 마이닝과 머신러닝 프로젝트에서 가장 중요한 단계이다.
- 분석 데이터 전처리의 데이터 품질은 가장 다루기 어려운 문제이지만 해결책은 현재 없어 보인다.
- 하지만 데이터 품질이 보장되지 않으면 데이터 분석결과는 사용하기 어렵다.(garbage in, garbage out)

### 데이터 전처리가 어려운 이유
- 우리의 현실...
    + 데이터를 처리하기 위해 여러 기술을 사용함
        * 그 기술의 종류만큼 데이터를 중복한다는 의미가 될 수 있으니 검토 대상이다.
    + Software Developer의 Data는 Storage, File이다.
        * Data는 Storage, File에 저장되는 것이지 Data = Storege, File은 아니다. Data로 현상과 사실이 설명될 수 있어야 한다.  
        * 그 사실들을 질의하여 언제든지 답을 얻을 수 있도록 정규화 해 놓은 것이 바로 Database다!
    + 데이터 처리는 SQL이 가장 효율적인데, SQL을 잘 사용하지 않는다.(선생님 의견)
        * 이유는 Data Engineer가 SQL 구사 능력이 부족하기 때문이라고 생각한다.
    + Requirement Analysis를 하지 않고 Design부터 한다.
        * 비즈니스 용어, 단어에 대한 고민은 없고 현재 Hot한 기술 용어에만 광적으로 집착한다.
    + 그 어느 누구도 데이터의 의미를 찾고 해석하지 않는다.
        * 이쁜 포장지(Data Visualization)와 내용물을 만든 공장에서 쓰인 기술(Big Data Storage)만 관심있지 내용물 (데이터->정보->지식)에 관심이 없다. 그래서 데이터를 보지 않는다.
    + 이렇게 해도 돌아가긴 한다. 단지 돈과 인력이 더 많이 들 뿐이다.
